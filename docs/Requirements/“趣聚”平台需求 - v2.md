# "趣聚平台"需求简述

文档说明:本案例仅用于暑期实践使用,请勿随意在网络上传播和转载,更不能用于其 他商业用途。

当前网络社会,各种活动信息分散难找,年轻人缺少轻松结识兴趣相同伙伴的线下渠道。 以兴趣为纽带的线下活动社交平台一方面能够汇聚各类活动,解决活动发布分散问题并方便 用户报名参与,两一方面促进以兴趣为基础交友,为大众提供真实无压力的线下社交载体。

本平台主要有三类角色:首先是平台运营方,其次是平台使用方,包括个人和商家。

#### (一)功能性需求

本次实践的"趣聚平台"包括以下几个方面功能:

# **1.** 用户信息管理

- 1)注册和登录:用户分为个人用户和商家用户两种角色。二者都需要使用邮箱和密码 完成注册,账户激活由平台发送激活链接;商家需要上传营业执照或营业凭证,后台审核通 过后获得商家身份。
- 2)个人资料管理:用户可维护个人资料,包括:头像、昵称(全平台唯一)、性别、 生 日、个性签名、兴趣标签等。 商家资料包括:商家名称、商家昵称、商家关注活动领域等。

# **2.** 活动全生命周期管理

# 1)活动创建

- ⚫ 用户创建活动:用户可以在平台内新建活动,新建时填写活动的基础信息,包 括:活动名称、标签、简介、时间、地点(地图选点)等内容。地点信息需要 支持通过地图进行选择。新建时完成活动参与设置,包括活动人数上限、报名 截止时间等。
- ⚫ AI 活动策划:用户可以输入活动主题或选择活动类型后,AI 自动生成活动基 础信息和参与设置等内容。生成结果支持用户逐项修改。
- ⚫ 活动模板与活动克隆:平台提供常用活动模板,例如运动健身、户外徒步、桌 游聚会、学习交流、公益活动、城市探索等类型。用户可以基于模板快速创建 活动。对于已经发布或历史完成的活动,发起人可以进行活动克隆,系统自动 复制活动基础信息和参与设置等信息,用户只需修改差异字段即可再次发布。
- ⚫ 草稿保存与提交:用户在活动创建过程中可以随时保存草稿,后期可以继续编 辑草稿。活动信息填写完成后,用户点击提交,系统根据活动内容和平台规则 进入自动审核或人工审核流程。

# 2)活动审核:

⚫ AI 自动内容安全审核:活动提交后,系统首先对活动相关信息的内容进行自 动审核。审核重点包括违法违规内容、低俗内容、危险活动等。AI 审核通过 的普通活动,系统可直接发布;对于存在风险但无法自动判断的活动,系统转 入人工审核。

- ⚫ 人工审核:对于报名人数超过 50 人的活动,或 AI 审核有问题活动系统进入 人工审核流程。人工审核人员后给出通过、驳回或要求修改的处理结果。
- 3)活动发现:用于帮助用户快速找到活动并查看活动详情。系统应提供首页信息流、 搜索页、高级筛选和地图模式等多种发现方式。其中地图模式必须有。
  - ⚫ 首页信息流:首页活动信息流包括推荐、最新和附近等 Tab。
  - ⚫ 搜索页:搜索页提供活动关键词搜索功能。
  - ⚫ 高级搜索:搜索页和首页可提供筛选器入口。用户可以按照活动类型、时间范围、 城市、费用和距离范围筛选活动。活动类型支持多选。
  - ⚫ 地图模式:首页附近 Tab 和搜索页提供地图模式入口。地图模式以地图点位形式展 示活动位置,用户可以查看当前位置周边的活动分布。
  - ⚫ 活动详情页:活动详情页集中展示活动的完整信息。详情页要显示活动状态(未开 始、报名中、截止、活动中等)。
  - 4) 活动报名:用于完成用户从查看活动到确认参与的转化过程。
  - ⚫ 报名校验:用户点击"立即报名"后,系统进行报名校验。例如,活动名额是否满额 校验,用户信誉、年龄等校验等。
  - ⚫ 报名确认:校验通过的报名,系统弹出报名确认窗口。窗口展示安全须知摘要,并 根据需要填写必要的报名信息。
  - ⚫ 取消报名:已报名用户可以在报名截止时间前取消活动。用户确认取消后,系统释 放名额,并在存在等待队列时自动递补队列第一位用户。
  - ⚫ 等待队列:当活动名额已满时,用户报告则进入等待队列,并显示当前等待人数。 一旦有名额,系统自动向队列第一位用户发送名额释放通知,该用户需要在限定时 间内确认报名,超时未确认则顺延至下一位。
  - 5) 活动签到:
  - ⚫ 扫码签到:活动发起人在活动开始前生成签到二维码。参与者到达活动现场后,通 过扫码完成签到。对于需要控制到场真实性的活动,可以结合位置校验,判断用户 是否处于活动地点附近。
  - ⚫ 签到管理:发起人可以在活动管理页查看报名用户和签到状态。活动结束后,签到 数据进入活动统计模块。
  - 6) 活动图文总结与评价
  - ⚫ 活动图文总结:活动结束后,发起人可以发布活动图文总结。系统可以对活动图片 进行 AI 分类识别,例如合影、场地、过程记录、物资、成果展示等,帮助发起人 快速整理素材。AI 分类结果需要由发起人进行人工确认,确认后再生成最终活动 总结。

⚫ 用户评价:活动结束后,参与者可以对活动评价。评价入口在活动结束后保留一定 时间,超过期限后自动消失。

# **3.** 好友社群

# 1) 好友形成和管理

- ⚫ 添加好友:用户可通过个人主页、活动参与者列表或扫描个人二维码发起好友 申请,对方可以同意和拒绝。已是好友或处于黑名单关系时,不可重复发送申 请。
- ⚫ 好友管理:用户可对好友列表设置备注、添加分组标签。用户删除好友后,双 方好友关系自动解除,好友备注和分组信息失效。
- ⚫ 关注关系:用户可以单向关注其他用户。双方互相关注后,关系升级为"好友"。 任一方取消关注后,互关和好友状态同步解除。

# 2) 兴趣小队生命周期

- ⚫ 兴趣小队建立:用户建立小队时需填写小队名称、兴趣标签、加入方式和人数 上限。加入方式分为公开加入和审核加入,小队人数有上限。创建者自动成为 队长。
- ⚫ 发现与加入:用户可按照小队名称或标签查找小队。公开小队可直接加入,审 核小队需要队长或管理员同意后加入。小队人数已满、已经解散或用户处于黑 名单关系时,不能加入。
- ⚫ 解散小队:只有队长可以解散小队。解散后所有成员退出,小队停止接受申请, 群聊和队内活动停止使用,小队不再显示在发现列表中。

# 3) 兴趣小队管理

- ⚫ 小队群聊:每个小队建立后自动生成群聊,成员加入或退出小队时同步加入或 退出群聊。群聊支持:队长和管理员发布群公告;成员使用"@"提醒指定成员; 队长和管理员可使用"@所有人";成员可参与群投票;成员可上传和查看群文 件。
- ⚫ 队内活动:队长和管理员可以发布队内活动,小队成员可以查看和报名。
- ⚫ 小队相册:成员可以上传活动照片,队长和管理员可以删除相册内容。
- ⚫ 积分榜:设置合理机制对每个小队成员进行积分,例如参与小队活动、发布动 态、动态被小队精选等。系统根据成员在小队中的积分生成排名,展示成员昵 称、积分和名次。

# 4) 兴趣小队权限管理

⚫ 小队设置队长、管理员和普通成员三类角色。队长:管理小队信息、成员、管 理员、公告、活动、文件和相册,并可解散小队。管理员:协助审核成员、管 理公告、活动、文件和相册,但不能解散小队。普通成员:参与群聊、投票、 上传文件和照片、报名活动以及查看积分榜。

# 5) 即时通讯

- ⚫ 消息发送:好友和小队成员之间支持发送文字、表情、图片和位置共享消息。
- ⚫ 消息操作:消息发送后显示未读或已读状态。用户可在发送后 2 分钟内撤回本 人消息,也可将消息转发至其他好友或小队群聊。

# **4.** 后台管理:

1) 管理员登录:管理员账号由系统预先创建,不提供注册功能。管理员可以修改本人 登录密码。

# 2) 用户管理:

- ⚫ 用户查询:管理员可以查看和查询平台中的个人用户与商家用户。管理员可以 查看用户基本信息、账号状态以及其发布的活动和创建的小队,但不得修改用 户的个人资料和密码。
- ⚫ 用户封禁与解封:管理员可以封禁或解封用户。封禁时必须填写原因,并明确 封禁期限;封禁后用户不能登录平台。解封后用户恢复正常登录和使用权限, 原有账号数据不受影响。
- ⚫ 商家申请审核:商家用户在提交商家身份申请后,由管理员进行人工审核,通 过或驳回。审核驳回时必须填写原因。
- 3) 活动管理:管理员可以查询平台中的全部活动,并按照活动状态进行筛选。对于已 经发布但存在违规问题的活动,管理员可以执行下架操作;下架时必须填写原因。 问题处理完成后,管理员可以恢复活动。

# 4) 小队管理

- ⚫ 小队查询:管理员可以查看和查询平台中的全部小队。管理员可以查看小队基 本信息、成员信息、队内活动及相关举报记录,但不得直接修改小队名称、标 签、公告和成员身份。
- ⚫ 小队停用与恢复:对于存在违规问题的小队,管理员可以执行停用操作,并填 写停用原因。小队停用后,不得新增成员、发布活动或新增队内内容,但原有 成员可以查看历史信息。违规问题处理完成后,管理员可以恢复小队。恢复后, 小队重新开放正常功能。

#### (二)非功能性需求

- 1)页面加载性能:在正常网络环境下,App 从完全关闭状态启动至登录页或首页可正 常操作的时间应小于 5 秒;Web 端从进入系统至登录页或首页可正常操作的时间应小于 5 秒。
- 2)后台接口性能:每一个后台业务接口的单次响应时间均应小于 2000 毫秒。AI 服务、 地图、短信等第三方接口,以及调用第三方接口的业务接口,不纳入该指标。
